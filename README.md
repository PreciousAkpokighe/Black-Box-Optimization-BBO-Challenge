Black-Box Optimisation (BBO) Capstone Project

Section 1: Project Overview

This project focuses on optimising a set of unknown functions using a Black-Box Optimisation (BBO) approach. Since the mathematical form of the functions is not provided, optimisation is performed iteratively by selecting input values, observing outputs, and refining subsequent inputs based on performance.
The project simulates real-world machine learning scenarios where systems must be optimised without direct knowledge of their internal structure.

Section 2: Input and Output

Number of functions: 8

Input domain: Continuous values between 0 and 1

Input dimensions: Vary per function (2D to 8D)

Output: Scalar value representing performance (fitness score)

Each function operates as a black box, meaning:

No access to gradients

No known function equations

Optimisation is based solely on input-output observations

Section 3: Challenge Objective

The objective of this project is to maximise the output values of eight unknown functions by identifying optimal input vectors within a bounded domain [0,1]^n.

Section 4: Technical Approach

Week 1: Initial Exploration
Inputs were distributed across the search space.

Purpose: broad exploration to understand general behaviour.

Outcome: Established baseline performance across all functions.

Week 2: Guided Optimisation
Inputs were adjusted using exploration vs exploitation principles.

Exploited high-performing regions (e.g., Function 5)

Explored new regions for poorly performing functions (e.g., Functions 4 and 6)

Strategy:

Outcome: Identified promising and weak regions in the search space.

Week 3: SVM-Inspired Optimisation
Introduced a classification-based perspective to guide input selection.

Approach:
Compared Week 1 and Week 2 outputs to label points as:

High-performing (positive class)

Low-performing (negative class)

Instead of training a full SVM (due to limited data), an SVM-inspired margin strategy was applied:

The better-performing point was treated as lying on the positive side of the decision boundary

The next input (Week 3) was generated by moving further in that direction, away from the lower-performing point

This mimics how an SVM maximises the margin between classes.

Why not a full SVM?
Each function only had 2 data points, which is insufficient to train a reliable soft-margin or kernel SVM.

Therefore, a heuristic approximation of SVM behaviour was used instead.

def svm_inspired_week3(w1, w2, y1, y2):
    if y2 > y1:
        best = w2
        worse = w1
    else:
        best = w1
        worse = w2

    direction = best - worse
